# 计算机视觉与模式识别实验报告

## 实验目的
- 熟悉图像处理的基本操作，OpenCV的基本使用；
- 通过对图像中不同元素的分割和统计，理解图像分割、形态学处理和像素统计的基本概念和方法；
- 学习如何处理图像中的重叠元素，并通过可视化展示结果。

## 实验内容
- image文件夹中有三幅图像，分别为:Al.jpg、Fe.jpg和P.jpg，代表某实验结果成像后三种化学元素的分布
- 1、统计每幅图中各化学元素的数目；
- 2、统计Al和Fe元素、Fe和P元素、Al和P元素重叠的数目，并且以图像的形式展示出两两元素的重叠情况；
- 3、统计Al、Fe和P元素三者重叠的数目，并以图像的形式展示出三种元素重叠情况。
- 扩展: 无论原始图像中的化学元素是何种颜色，均可实现本实验目标。

## 实验过程

### 方法一：基于二值化与形态学处理的元素统计

本方法是实验的初始方案，通过将图像二值化后检测连通域来统计元素数目。

#### 1. 二值化处理

将输入图像从BGR色彩空间转换到HSV色彩空间，利用明度(V)通道进行阈值分割。设置阈值范围为V>50，将元素区域（彩色）与背景（黑色）分离。

```cpp
Mat hsv, mask;
cvtColor(image, hsv, COLOR_BGR2HSV);
inRange(hsv, Scalar(0, 0, 50), Scalar(180, 255, 255), mask);
```

#### 2. 形态学处理与连通域分析

**遇到的问题：** 直接对二值图像进行轮廓检测时，由于元素之间存在紧邻或重叠的情况，相邻元素会被识别为同一个连通域，导致统计结果偏小。

**解决方案：** 使用3×3十字形结构元素对二值掩码进行轻度腐蚀(Erode)操作。十字形核的"切断"能力较为轻柔，能够断开边缘仅相连1-2像素的相邻元素，同时不会过度侵蚀小型元素。随后使用`connectedComponents`进行8连通域标记，统计独立元素数量。

```cpp
Mat kernel = getStructuringElement(MORPH_CROSS, Size(3, 3));
erode(mask, processedMask, kernel);
int count = connectedComponents(processedMask, labels, 8) - 1;
```

#### 4. 重叠检测

对于两两元素重叠的情况，通过将对应元素的二值掩码进行按位与(`&`)运算得到重叠区域，再统计重叠区域的连通域数量。

### 方法二：基于3×3块中心点检测的尝试

在观察到每个元素呈现为3×3像素块的规律后，尝试了另一种思路：以3为步长遍历图像，取每个3×3块的中心像素判断该位置存在哪些元素。

然而，该方法存在缺陷：由于步长固定的遍历方式，且三个元素图像的叠加位置可能存在偏移，导致统计结果出现数量级偏差，数据严重失真。因此该方法被放弃。

## 实验小结

### 1. 方法一的改进与成效

在实验初期，直接使用轮廓检测或连通域标记时，遇到的主要问题是**紧邻/重叠元素的合并**。相邻的元素会被当作同一个连通域，导致统计数目远小于实际值。

通过采用**3×3十字形结构元素进行轻度腐蚀**的方法，一定程度上缓解了这一问题。十字形核的优势在于：
- 能够切断边缘仅相连1-2像素的紧邻元素
- 相比矩形核，保留更多的元素面积信息
- 相比椭圆核，分离效果更加干净

改进后的结果呈现出合理的数据关系：
- 单独元素数量：Al(4848) > Fe(2677) > P(1287)
- 两者重叠数量均在合理范围内
- 三者重叠数量(27)小于任意两者重叠数量

### 2. 方法二的教训

方法二尝试利用元素为3×3像素块的规律，但忽略了关键问题：**三个元素的图像并非完美对齐**，以固定步长遍历会导致中心点判断出现偏差，最终统计结果完全失真。

这说明在图像处理中，**不能想当然地假设理想化的规律**，需要充分考虑实际图像的复杂性和误差。

### 3. 实验拓展的思考

任务要求"无论原始图像中的化学元素是何种颜色，均可实现本实验目标"。当前方法基于HSV色彩空间的明度(V)通道进行二值化，理论上只要求元素颜色足够深（V>50）即可正常工作。如果元素颜色较浅或背景并非纯黑，可能需要采用其他分割策略如聚类分析或自适应阈值。

### 4. 进一步优化的方向

当前方法仍有改进空间：
- 可以尝试不同形状和大小的形态学核，找到最佳的分离效果
- 重叠区域的元素分离仍可能不够完美，可考虑分水岭算法等更精细的分割方法
- 没有对图像左上角和左下角的图例和比例说明进行除了，这部分会被统计入元素数量
- 可引入机器学习的方法进行实例分割
